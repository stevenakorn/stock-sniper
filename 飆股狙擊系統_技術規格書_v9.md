# 飆股狙擊系統 v9.0 — 技術規格書（AI 接手用）

> 單檔 HTML 台股掃描系統。給接手的 AI 或開發者：讀完這份就能安全改動，不用重新逆向整份程式碼。
> 對應檔案：`飆股狙擊網頁版.html`（約 172 KB，單檔含全部 CSS/JS）+ `manifest.json` + `sw.js` + `icon-192.png` + `icon-512.png`。

---

## 1. 這是什麼

台股盤後選股工具。使用者按「盤後分析」→ 系統掃全市場股票 → 用三種策略篩出訊號 → 給 1~5 星評分 → 依星級排序顯示。附帶：個股詳情（K 線圖 + 籌碼）、產業資金流向雷達、融資融券查詢、盤中即時掃描、**訊號回測**、**大盤濾網**。

資料來源是 FinMind API，透過一個 Vercel/Flask 後端 proxy 轉發（避開 CORS + 隱藏 token）。

## 4. 掃描資料流（核心）

```
使用者按「盤後分析」→ startScan()
  → 取 STOCK_LIST（全市場代號），每批 35 檔並行
  → 每檔 fetchStockData(id)  // 90天日K，有記憶體快取
      → checkBreakout / checkConsolidation / checkAccumulation  // 三策略，任一命中即為訊號
  → 對命中的再 fetchChipData(id)  // 三大法人+融資+分點券商
      → analyzeChip() 算籌碼分 chipScore(0~5)
  → 訊號存入 hits[]，蓋 result.date = todayISO()
  → calcStars() 算 1~5 星 → renderHits() 排序渲染
  → localStorage "hits_today" 存當日訊號
  → 掃描結束：archiveSignals() 歸檔 + renderBacktest() 更新回測面板
```

三策略判斷（`checkBreakout` 等）回傳的訊號物件欄位：
`{ id, name, time, date, type, label, close, changePct, volRatio, detail, chipScore, foreignNet5, trustNet5, totalNet5 }`
其中 `type` ∈ `breakout` / `consolidation` / `accumulation`。

**FinMind 欄位注意**：`Trading_Volume` 單位是「股」，籌碼張數需 `/1000` 換算成「張」（`analyzeChip` 與 `drawInstChart` 都已處理）。`max`/`min` 對映 high/low。

## 5. 星級計算 `calcStars(h)`

```
量價分(0~3)：volRatio ≥3→+3, ≥2→+2, ≥1.5→+1
籌碼分(0~2)：chipScore ≥4→+2, ≥2→+1
漲幅分(0~1)：|changePct| ≥3 → +1
策略加成  ：breakout → +1（上限5）
大盤濾網  ：MKT.filterOn && MKT.loaded && MKT.bearish → -1  ★v9新增
最終       ：clamp 到 1~5
```

**重要**：歸檔用的是 `calcStarsRaw()`（不含大盤濾網懲罰），因為回測基準星級要固定、不能隨當下大盤變動。`calcStars()` 才含濾網懲罰，用於即時顯示。改星級規則時兩個都要同步。

## 6. 回測系統（v9.0，本次重點）

### 資料歸檔
- 每次掃描/即時掃描結束 → `archiveSignals()` 把 `hits[]` 寫入 localStorage `sniper_history_v1`。
- 結構：`{ "YYYY-MM-DD": { "股號": {訊號摘要, stars: calcStarsRaw, bt: 回測結果 or null} } }`
- 保留上限 60 天（`HIST_MAX_DAYS`），超過砍最舊。
- 歸檔時保留既有 `bt` 結果（`prev.bt`），不會被覆蓋。

### 回測計算 `runBacktest()`
- 收集所有「訊號日 < 今天」且 `bt` 為 null 或 `bt.pending` 的訊號。
- 每檔抓 `fetchStockData`（記憶體快取，同股不重打），批次 8 檔。
- **進場假設（v9.0 已改為隔日開盤）**：
  - `entryIdx = 訊號日索引 + 1`，`進場價 entry = d.open[entryIdx]`（隔日開盤）
  - 理由：使用者盤後才看到訊號，當日已收盤，最快隔天開盤才能進場。
  - `+N 日報酬 r(n) = (d.close[entryIdx + n - 1] - entry) / entry × 100`
    （即進場當天算第 1 日，取當日收盤）
  - `bt = { entry, entryBasis:"nextOpen", r1, r3, r5, r10 }`
  - 隔日開盤資料未出 → 整筆 `{pending:true}`，下次再算。
  - `r10 === null`（未滿10日）→ `bt.pending = true`，之後補算。
  - 訊號日超過 85 天仍找不到（停牌/下市）→ `bt = {expired:true}`，不列入統計。

> **若要改回「訊號日收盤進場」**：把 `entryIdx = idx + 1` 改回 `idx`、`entry = d.open[entryIdx]` 改回 `d.close[idx]`、`r(n)` 改回 `d.close[idx+n]`，並移除 `entryBasis`。同時改明細表底部說明文字與 `<th>進場價</th>` 註記、CSV 表頭 `進場價(隔日開盤)`。

### 統計與渲染 `renderBacktest()`
- `collectBtRows()` 撈出所有已完成（非 expired、r1 有值）的訊號。
- 勝率主判準：**+5 日報酬 > 0**（不足5日 fallback r3→r1）。
- 產出：4 張總覽卡（已回測數/勝率/均酬5日/均酬10日）+ 各策略勝率表 + 各星級勝率表 + 明細表（最近 80 筆，可點列開個股詳情）。
- CSV 匯出 `exportBacktestCSV()`：BOM + CRLF，Excel 開中文不亂碼。

**第一次上線回測是空的**，正常。要每天按掃描累積歷史，隔日後按「↻ 更新回測」才有數據。

## 7. 大盤濾網（v9.0）

- `MKT` 物件持有狀態：`{ filterOn, bearish, loaded, close, ma20, ma60, chgPct, date }`。
- `loadMarketFilter()`：抓加權指數 TAIEX（`fetchIndexData`，120天，有日快取），`close < ma20` 即 `bearish=true`。
- 空頭時：header 晶片變綠、`calcStars` 全體 -1 星、log 警示。
- 開關：header 下方 `#mktFilterToggle`，狀態存 localStorage `mkt_filter_on`（預設開，`"0"` 為關）。
- `stock_id=TAIEX` 走後端 `/api/stock_price`。若後端不支援 TAIEX，晶片顯示「大盤資料暫無法取得」，濾網自動失效（`MKT.loaded=false` 時不懲罰），不影響其他功能。

## 8. 快取層（v9.0）

兩級快取，都在最上方模組：

| 類型 | 變數/前綴 | 存活 | 用途 |
|---|---|---|---|
| 記憶體 | `memCache` (Map) | 盤中 5 分、盤後 6 小時 | 股價 `px_*`、籌碼 `chip_*`；掃描/詳情/回測共用，同 session 同股不重打 |
| localStorage | `csn9_*` | 當日（跨日自動失效） | 股票清單 `stocklist`、資金流向 `moneyflow`、大盤 `taiex`、融資查詢 `mq_*` |

- `memGet/memSet`、`lsCacheGet/lsCacheSet` 是統一入口。
- localStorage 配額爆掉時 `lsCacheSet` 會清掉所有 `csn9_` 前綴後重試一次。
- 啟動時 `cleanOldCache()` 清掉非今日的日快取。

## 9. 個股詳情 + 圖表（v9.0 強化）

- `searchStock()`：輸入代號/名稱 → 抓股價+籌碼 → 渲染技術面/均線/籌碼面/五大券商 → **注入兩張 Canvas 圖**。
- `drawKline(d)`：近 60 根日 K（紅漲綠跌）+ MA5(琥珀)/MA20(藍) + 量能，DPR 縮放清晰。
- `drawInstChart(chip)`：外資+投信每日合計買賣超柱狀圖（近 20 日，`/1000` 轉張）。無資料自動隱藏。
- `openStockDetail(id)`：訊號卡片、回測明細列點擊 → 帶入代號查詢並捲動到詳情。
- Canvas 繪圖包在 `requestAnimationFrame` 內，確保 DOM 已 render 拿得到寬度。

## 11. 已知限制與注意事項

1. **回測是「紙上回測」**：無滑價、無手續費、無漲停買不到的模擬，進場假設為隔日開盤全額成交。僅供策略相對比較，非實際績效。
2. **分點券商（broker）資料**：TWSE 對分點資料有限制，`fetch_broker_trading` 可能常態回空，`drawInstChart` 與五大券商區塊會自動隱藏，屬正常。
3. **TAIEX 大盤資料**：依賴後端支援 `stock_id=TAIEX`。若不支援，濾網靜默失效。
4. **死程式碼**：`onScanSettingChange`/`startPanelScan`/`setMarket` 引用的產業選單 UI（`industrySelect`/`btnPanelScan`/`scanHint`/`.market-btn`）在此版 body 不存在，這些函式不會被觸發。已加 null 防呆，若未來要恢復產業選單需補回對應 HTML。
5. **API 額度**：FinMind Sponsor tier 有速率限制。快取層已大幅減少重複請求；回測一次最多處理 200 筆、批次 8、間隔 120ms。
6. **顏色慣例**：全站台股慣例 **紅漲綠跌**。CSS 變數 `--up:#f0424e`（紅）、`--down:#22a06b`（綠）。改配色勿把這兩個對調。
